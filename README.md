# Autotest_And_SQL
# README к автотесту test_scooter_order.py
# СУТЬ КОДА: 
- Автотест на проверку ответа от сервера статусом 200 Ok при выполнении GET-запроса на получение заказа по его трэк-номеру: документ test_scooter_order.py
# ОПИСАНИЕ: 
- Автотест написан, опираясь на мой референсный код с фикстурой для "ловли" и фиксации в тестировании непостоянных / уникальных значений. Создан при изучении питона факультативно, структура файлов для автоматической проверки и сама структура кода автотеста несколько отличаются от того, чему нас учили в материалах Практикума для автоматизации тестирования: здесьбыло принято решение делать все в 1-м документе, чтобы самому не запутаться в ссылках на разные доки и переменные в них + по-другому пока не знаю, как работать с фикстурой, когда у нас вариабельная сущность трэк-номера (уникальная от post-запроса к post-запросу на создание заказа); новый трэк-номер нужно при каждом выполнении теста подхватывать и фиксировать для осуществления успешной проверки на код 200 в получении заказа по трэк-номеру. Следовательно -> документы с фиксированной информацией для этого автотеста заранее организовать довольно проблематично, в отличие от автоматизации проверок фиксированных параметров и их значений. 
- Используемые источники для автоматизации: Хабр -> материалы про документацию pytest, фикстуры pytest; книга "Укус питона" К. Сварупа в переводя В. Смоляра
# УСЛОВИЯ ЗАПУСКА АВТОТЕСТА:
- Для корректного функционирования теста должны быть установлены пакеты requests и pytest в среде venv (pip install requests / pip3 install requests из под venv; pip install pytest / pip3 install pytest из под venv) 
- Запуск теста выполняется командами pytest -v и pytest test_scooter_order.py в терминале, также автоматически подхватывается во вкладке Testing и в самой консоли пайтестом (см. приложенные png)
